feat: Inline Diagram Generation

We have successfully implemented inline diagramming and interactive notes features in PDFInsight. The academic tutor can now explain complex visual structures, workflows, networks, and UML designs dynamically.

 Key Accomplishments

# 1. Multilevel Classification Pipeline
- Scope Detection (`detect_diagram_scope`): When the "Add Diagrams" checkbox is enabled, PDFInsight uses a fast LLM classification call to decide if the query has scope for a diagram. If the query contains heuristic keywords (e.g., `workflow`, `process`, `flowchart`, `loop`, `diagram`, `pipeline`, `architecture`) but the LLM returns "No", it gracefully maps it to Ambiguous.
- Interactive Scope Dialog: Ambiguous queries prompt the user directly in the chat with interactive "Yes, include a diagram" and "No, text only" buttons. Choosing a button reruns the app and triggers the correct generation path.
- Tool Selection (`select_diagram_tool`): If diagram scope is positive, the system determines the optimal tool for the prompt structure:
  - Mermaid: Flowcharts, state transitions, sequential loops, simple sequence diagrams.
  - Graphviz: Network topologies, complex node-edge connections, mathematical graphs.
  - PlantUML: Software engineering UML (classes, structures, sequence lifecycles).
  - D2: High-level systems architecture, org charts, directory/tree layouts.

# 2. High-Performance Inline Rendering
Fenced code blocks inside LLM responses are parsed dynamically and rendered using zero-dependency, web-based layouts:
- Mermaid & D2: Rendered dynamically using secure browser-native WebAssembly (WASM) and ESM loaders embedded inside clean Streamlit HTML iframes.
- Graphviz: Rendered using Streamlit's built-in `st.graphviz_chart()` engine.
- PlantUML: Encoded via a custom Python zlib-deflate translation algorithm and requested dynamically as an SVG from the official public PlantUML rendering service.

# 3. Response Editing & Annotations
Every assistant response features an inline `📝 Edit Response & Add Notes` expander:
- Users can directly modify the markdown content of the response and save changes.
- Users can upload local images (`.png`, `.jpg`, `.jpeg`) to annotate the response. Uploaded images are stored under `uploaded_images/` and appended in the chat layout.

# 4. PDF Conversation Export with Rendered Diagrams
We updated `core/export.py` to compile the entire conversation history:
- Diagram blocks are intercepted, compressed, URL-safe Base64 encoded using raw DEFLATE, and sent to Kroki.io to retrieve high-quality SVG vector graphics.
- Uploaded custom annotation images are referenced via their local file system paths.
- Temporary files are rendered into the final multipage PDF using PyMuPDF and cleaned up immediately after compilation.
