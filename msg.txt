feat: build workspace UI, modularize chat, and improve UX

- Modularization: The monolithic `ui/chat.py` had grown excessively large causing maintenance difficulties and critical indentation errors during development. To establish a more scalable and resilient architecture, `chat.py` was structurally refactored into three distinct, focused modules:
  1. `ui/chat.py`: Now purely serves as the Streamlit layout and presentation layer.
  2. `ui/diagrams.py`: Centralizes all complex diagram rendering (Mermaid, Graphviz, D2, PlantUML).
  3. `ui/chat_handlers.py`: Encapsulates the heavy-lifting logic for Qdrant vector queries and Groq LLM invocations.
- Enhanced Study Workflow: The application's core functionality was expanded from a basic chat interface into a fully featured environment by adding a real-time Markdown Notes Workspace (`ui/editor.py`). This allows users to actively build study guides side-by-side with the chat.
- Data Capture & Retention: Added a seamless `➕ Append to Notes` button on all LLM responses, allowing users to effortlessly transfer generated insights directly into their Master Notes workspace.
- Export Capabilities: Implemented a comprehensive PDF export engine (`core/export.py`), leveraging `markdown-pdf` to convert the dynamic Master Notes into a highly formatted, downloadable study guide.
- UX Polish & Responsiveness: Replaced clunky UI toggles with sleek, native-feeling `❮` and `❯` panel control buttons to maximize screen real estate when reading dense text. Resolved native Streamlit limitations by dynamically spacing the `st.chat_input` so it naturally anchors to the bottom of the column structure, mimicking industry-standard conversational UX.
- Clean Repository State: Configured `.gitignore` to prevent local PDF caches from polluting the repository history, and purged existing redundant dummy files.
